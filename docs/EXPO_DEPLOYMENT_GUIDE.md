# คู่มือการอัปเดตและสร้างแอปพลิเคชัน (Expo / EAS)

คู่มือนี้อธิบายขั้นตอนการอัปเดตแอปพลิเคชัน Expo (Mobile BMR) ทั้งในกรณีที่มีการแก้ไขเล็กน้อย (Over-The-Air Update) และกรณีที่ต้องสร้างตัวแอปใหม่ทั้งหมด (EAS Build)

---

## 1. การอัปเดตแอปเล็กน้อย (EAS Update / OTA)
**เมื่อไหร่ที่ควรใช้:** 
ใช้เมื่อมีการแก้ไขโค้ดที่เกี่ยวกับ UI, สี, ขนาดข้อความ, ฟังก์ชัน JavaScript ภายในแอป หรือแก้บั๊กทั่วไป (ไม่เกี่ยวข้องกับการเพิ่มไลบรารี Native ใหม่ๆ หรือการเปลี่ยนเวอร์ชันของ Expo SDK)

**ข้อดี:** 
ผู้ใช้งานแอปพลิเคชันจะได้รับอัปเดตอัตโนมัติเมื่อปิดแล้วเปิดแอปใหม่ โดยไม่ต้องไปดาวน์โหลดแอปใหม่จาก Play Store/App Store หรือติดตั้ง APK ใหม่

### ขั้นตอนการอัปเดต (OTA Update)
1. เปิด Terminal และเข้าไปที่โฟลเดอร์ของโปรเจกต์
   ```bash
   cd mobile-BMR
   ```
2. พิมพ์คำสั่ง EAS Update พร้อมระบุข้อความกำกับการอัปเดต
   ```bash
   eas update --branch production --message "ปรับขนาดปุ่มหน้าเลือกสาขาและแก้ UI"
   ```
   *(หมายเหตุ: `--branch production` คือชื่อ branch ที่ผูกไว้กับแอปตัวจริง หากใช้ตัวทดสอบอาจใช้ `--branch preview` ตามที่ตั้งค่าไว้ใน `eas.json`)*
3. รอจนกระบวนการอัปโหลดเสร็จสิ้น ระบบจะขึ้นลิงก์สำหรับดูสถานะบนหน้าเว็บ Expo

---

## 2. การสร้างแอปพลิเคชันเพื่อติดตั้งนอกสโตร์ (ไฟล์ APK สำหรับ Android)
**เมื่อไหร่ที่ควรใช้:** 
- เมื่อต้องการให้พนักงานหรือทีมงานติดตั้งแอปโดยตรงผ่านไฟล์ `.apk` โดยไม่ต้องผ่าน Google Play Store
- เมื่อมีการแก้ไข Native Code, เปลี่ยน App Icon หรือเปลี่ยนเวอร์ชันแอป

### ขั้นตอนการ Build ไฟล์ APK
1. **อัปเดตเวอร์ชันของแอป (ถ้าจำเป็น):** 
   เปิดไฟล์ `app.json` และทำการปรับตัวเลขเวอร์ชัน เช่น
   ```json
   "version": "1.0.2",
   ```

2. **เปิด Terminal และเข้าไปที่โฟลเดอร์ของโปรเจกต์**
   ```bash
   cd mobile-BMR
   ```

3. **รันคำสั่งสร้าง Build เป็น APK:**
   ระบบได้ตั้งค่า Profile พิเศษไว้สำหรับสร้าง APK ไว้แล้ว ให้ใช้คำสั่ง:
   ```bash
   eas build --platform android --profile production-apk
   ```
   *(Profile `production-apk` ถูกตั้งค่าใน `eas.json` ให้สร้างไฟล์ .apk แจกจ่ายภายในโดยเฉพาะ)*

4. **ดาวน์โหลดและติดตั้ง:**
   - หลังจากรอจนเสร็จสิ้น (ประมาณ 5-15 นาที) ระบบจะแสดงลิงก์ดาวน์โหลด `.apk` 
   - ส่งลิงก์นี้หรือไฟล์ `.apk` ให้กับพนักงาน
   - **สำคัญ:** ผู้ติดตั้งจะต้องเข้าไปตั้งค่าเครื่องให้ยอมรับ **"การติดตั้งแอปจากแหล่งที่ไม่รู้จัก" (Install unknown apps)** ก่อน จึงจะสามารถติดตั้งไฟล์ APK ได้

---

## 3. การสร้างแอปพลิเคชันขึ้นสโตร์จริง (App Bundle / IPA)
**เมื่อไหร่ที่ควรใช้:** 
เมื่อต้องการอัปโหลดตัวแอปขึ้น Google Play Store (ไฟล์ `.aab`) หรือ Apple App Store

### ขั้นตอนการ Build:
- **สำหรับ Android (Google Play Store):**
  ```bash
  eas build --platform android --profile production
  ```
- **สำหรับ iOS (Apple App Store):**
  ```bash
  eas build --platform ios --profile production
  ```

---

## 4. สรุปคำสั่งที่ใช้งานบ่อย

| คำสั่ง | หน้าที่ |
| --- | --- |
| `eas login` | ล็อกอินเข้าสู่ระบบบัญชี Expo |
| `eas update --branch production -m "..."` | ปล่อยอัปเดตหน้าตาแอป/โค้ด ไปให้ผู้ใช้ทันที (ไม่ต้องลงแอปใหม่) |
| `eas build -p android --profile preview` | สร้างไฟล์ติดตั้ง `.apk` สำหรับแอนดรอยด์เพื่อทดสอบ |
| `eas build -p android --profile production` | สร้างไฟล์ `.aab` เพื่อเอาขึ้น Play Store |
| `npx expo start -c` | รันแอปในโหมดพัฒนา พร้อมเคลียร์แคช (แนะนำให้ทำทุกครั้งที่อัปเดตไฟล์ใหญ่ๆ) |
